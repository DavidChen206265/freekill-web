## Phase 9 Summary

The catalog/encyclopedia layer (browse all generals/cards, filter by source fields, full detail pages with audio/stats/favorites/same-name navigation, ban-scheme export) remains essentially **not restored**. Only an in-game `GeneralDetailModal` (focused player-skill inspector) and the gameplay `cardname` Interaction control exist.

**One correction vs. a naive read:** P9-017 — the *gameplay* `cardname` skill-interaction control IS restored (Dashboard `ComboInteraction`), even though the *catalog* CardNamesBox flick-picker is not. This is the only nuance vs. "entirely absent".

**High-risk false-confidence note:** the source catalog Lua helpers (`GetGeneralDetail`/`SearchGenerals`/`FilterAllGenerals`/`GetAllGeneralPack`/`GetSameGenerals`) ARE copied into the public Lua tree but are NOT exposed through the React VM bridge (`clientVm.ts` exposes only readCards/readGenerals/playerSkills/gameSummary/choose/poxi/choice). Their presence in assets does not mean the catalog UI works.

This whole phase is a candidate "M6 encyclopedia" milestone if catalog browsing is desired; it does not affect core in-game play.
