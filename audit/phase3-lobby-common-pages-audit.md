## Phase 3 Summary

The web lobby restores a minimal operational path: connect to gateway, refresh room list, create a basic room, join/observe (with password prompt), show online counts, plain lobby chat. Most source Common/Lobby pages remain not restored: server browser, lobby filters, MOTD/bulletin, preferred/custom pages, full create-room settings, mode/package/ban settings, profile/avatar/password, audio/control/UI/background/language settings, package/resource managers, tutorial, modes/about, rich chat/danmu, widget system.

**Key change since M2:** P3-005 passworded observe → **successfully restored** (shared `promptPassword` for join + observe, M3). P3-024 gained lobby Heartbeat echo (M3) + VmDebugPanel comms log (M4-V).

Highest-risk gaps: create-room fixed to one mode with no dynamic `_game`/`_mode` settings (P3-010/011/012); no filter/preferred/custom navigation (P3-006/009); no package/resource management or server discovery (P3-002/019/020).
