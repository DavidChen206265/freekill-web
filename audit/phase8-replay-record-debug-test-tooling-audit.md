## Summary

- Replay and local record management remain **not restored**. Source has three replay pages, SQLite game history, `.fk.rep` import/export/delete/comment, replay timeline controls, and a `QThread` replayer. Web has no replay route, no browser record DB, no `.fk.rep` parser/player, and `CppClient:saveRecord/saveGameData` are empty stubs.
- End-game review is **simplified**: web restores the game-over summary table and return-to-room/lobby actions, but omits continue-game, honor, save replay, and bookmark replay.
- `lua/ui_emu/**` is **successfully copied** into the web VM mount with no content diff (only line-ending warnings) — the basis for interactive request handling.
- `packages/test` is **partially restored**: Lua copied; QML test components and test media omitted; two skill files diverge.
- **NEW (M4-V):** structured diagnostics — `diag/log.ts` (ring buffer + level control), `diag/notifyCommands.ts` (command classifier + unhandled-detector), `VmDebugPanel.tsx` (notifyUI comms log/export/warnings).
